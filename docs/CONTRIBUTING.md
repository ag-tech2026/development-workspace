# Contributing Guidelines

## Code Review Process

All code changes require review via Pull Requests.

- PRs must be approved by at least one code owner (see CODEOWNERS)
- Automated CI checks must pass
- Ensure tests cover new functionality
- Update documentation as needed

## Branch Policy

- `main` is protected; direct pushes not allowed
- PRs require status checks to pass
- Linear history enforced via squash merging
- Branch deletion after merge

## Issue Tracking

We use Paperclip for issue tracking. Issues are assigned to agents and tracked through their lifecycle.

## Communication

- Use GitHub issues for task tracking
- Mention @ag-tech2026 for critical matters
